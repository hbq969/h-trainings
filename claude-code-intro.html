Skills vs SubAgent

Skills

  • 工具和流程组合
  • 用于复用
  • 定义"怎么做"

SubAgent

  • 独立任务执行单元
  • 执行"做什么"
  • 并行处理子任务

⚠️ 注意

  • 子 agent 之间没法协调沟通
  • 复杂任务建议用规划模式

经验总结

避免踩坑的 7 条核心经验

01

先写 CLAUDE.md，再写代码

模型上下文理解能力很强，但它得先知道你的规则。把技术栈、目录结构、编码习惯全塞进 CLAUDE.md 中。

02

别着急写业务，先让它画蓝图

用 plan 模式，让 AI 先输出文件结构、模块划分、数据流向。审完蓝图再开工，返工率直接降 80%。

03

一次只干一件事

千万别让它一口吃一个胖子。把需求拆成小任务一个个做，AI 在小范围里的代码质量极高。

07

经验总结 · 续

04

先写测试，再写实现

让 AI 自己先出测试用例，等于给它定好及格线。测试本身就是最好的文档。

05

Code Review 永远不能省

AI 写的代码 90% 能用，但那 10% 的坑藏的很深。每次花两分钟扫一眼关键逻辑。

06

用 MCP 和 Skills 给它立规矩

把项目的最佳实践封装成 MCP 或 Skills。AI 直接按你的标准来写代码。

07

该重置上下文就重置

一旦发现它开始答非所问，果断重置。一个干净的上下文比又臭又长的对话高效 10 倍。

Vibe Coding 真不是偷懒，它是一套全新的工程能力。会用的人效率起飞，不会用的人只会觉得"AI也不太行啊"。

核心就一句话：你得非常清楚地知道自己要什么，AI 才能帮你快速完成落地。

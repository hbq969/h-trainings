项目记忆文件 CLAUDE.md

定义项目规则和上下文

# Project Rules

## 技术栈
- React + TypeScript
- Node.js + Express

## 代码风格
- ESLint + Prettier
- 命名规范: PascalCase for components
- 使用 async/await 替代 Promise.then

## 目录结构
- src/components/ - 组件目录
- src/hooks/ - 自定义 Hooks
- src/utils/ - 工具函数

## 注意事项
- 所有 API 调用需要错误处理
- 组件必须有 TypeScript 类型定义

经验总结 (上)

避免踩坑的核心经验

01

先写 CLAUDE.md，再写代码

把技术栈、目录结构、命名编码习惯全塞进 CLAUDE.md 中去，没有 CLAUDE.md 的 Vibe Coding，就是在开盲盒。

02

别着急写业务，先让它画蓝图

用 plan 模式，让 AI 先输出文件结构、模块划分、数据流向、待办任务列表。审完蓝图再开工，返工率直接降 80%。

03

一次只干一件事

把需求拆成一个个小任务，一个个的做。AI 在小范围里的代码质量极高，但上下文一长就容易丢细节还慢费 token。

04

先写测试，再写实现

让 AI 自己先出测试用例，等于给它定好及格线。然后让它照着测试写代码，一次通过率高的不是一点半点。

13

经验总结 (下)

避免踩坑的核心经验

05

Code Review 永远不能省

AI 写的代码 90% 能用，但那 10% 的坑藏的很深。每次写完一个小任务花两分钟扫一眼关键逻辑，能省你两个小时的 Debug。

06

用 MCP 和 Skills 给它立规矩

把你项目的最佳实践封装成 MCP 或 Skills。这不是教人写代码，是教 AI 按你的规矩来写代码。

07

该重置上下文就重置，别硬撑

一旦发现它开始答非所问，或者忘了你之前定的约束，果断重置。一个干净的上下文，比一个又臭又长的对话高效 10 倍。

Vibe Coding 真不是偷懒，它是一套全新的工程能力。会用的人效率起飞，不会用的人只会觉得"AI也不太行啊"。

核心就一句话：你得非常清楚地知道自己要什么，AI 才能帮你快速完成落地。
